# 轨道交通乘务与交路管理系统架构方案

## 1. 推荐技术路线

- 前端：React + TypeScript + Tailwind + React Router + Zustand + TanStack Query
- 后端：NestJS 或 Express + TypeScript
- 数据库：PostgreSQL
- 实时层：WebSocket
- 任务编排：BullMQ + Redis
- 文件存储：MinIO / OSS / S3
- 文档解析：`xlsx`、`mammoth/docx`、`pdfplumber/pdf-parse`、OCR 适配层

推荐把全局状态和服务端查询状态分开：

- Zustand 只放跨页面共享且高频变化的状态：当前操作人、当前值乘任务、实时连接状态、选中车次、全局筛选条件
- TanStack Query 负责接口数据：主班表列表、历史车次分页结果、导入任务详情、运行图版本列表

## 2. 模块拆分

- `dashboard`：全局总览，对应 UI 中的“全局交路图”
- `running-graph`：运行图与全线车次态势
- `attached-route`：随车交路图，跟踪单次任务“一回”
- `master-schedule`：主班表导入、标准交路维护
- `history-trips`：历史车次查询与回放
- `import-center`：统一文件上传、解析、校验、入库

## 4. 前端全局 Store 设计

```ts
import { create } from "zustand";

type ConnectionStatus = "CONNECTING" | "ONLINE" | "OFFLINE";

interface HistoryQuery {
  operatorName?: string;
  trainNo?: string;
  scheduleVersionId?: string;
  date?: string;
  routeId?: string;
  tripId?: string;
  source?: "master-schedule" | "attached-route" | "dashboard";
}

interface ImportCenterState {
  currentJobId?: string;
  jobsById: Record<string, ImportJob>;
}

interface RealtimeBoardState {
  connectionStatus: ConnectionStatus;
  lastSyncAt?: string;
  vehiclesById: Record<string, RealtimeVehicleState>;
}

interface AppStoreState {
  operatorContext?: OperatorContext;
  currentDuty?: CurrentDutyContext;
  selectedTripId?: string;
  selectedTrainNo?: string;
  activeScheduleVersionId?: string;
  historyQuery: HistoryQuery;
  importCenter: ImportCenterState;
  realtimeBoard: RealtimeBoardState;
  setOperatorContext: (payload: OperatorContext) => void;
  setCurrentDuty: (payload?: CurrentDutyContext) => void;
  setSelectedTrip: (payload?: { tripId?: string; trainNo?: string }) => void;
  setActiveScheduleVersion: (scheduleVersionId?: string) => void;
  patchHistoryQuery: (patch: Partial<HistoryQuery>) => void;
  upsertImportJob: (job: ImportJob) => void;
  setImportCurrentJob: (jobId?: string) => void;
  setRealtimeConnectionStatus: (status: ConnectionStatus) => void;
  mergeRealtimeVehicles: (items: RealtimeVehicleState[]) => void;
  archiveCurrentTrip: (actualArrivalAt: string) => void;
  resetDutyContext: () => void;
}

export const useAppStore = create<AppStoreState>((set, get) => ({
  historyQuery: {},
  importCenter: { jobsById: {} },
  realtimeBoard: { connectionStatus: "OFFLINE", vehiclesById: {} },

  setOperatorContext: (payload) => set({ operatorContext: payload }),
  setCurrentDuty: (payload) => set({ currentDuty: payload }),
  setSelectedTrip: (payload) =>
    set({
      selectedTripId: payload?.tripId,
      selectedTrainNo: payload?.trainNo,
    }),
  setActiveScheduleVersion: (scheduleVersionId) =>
    set({ activeScheduleVersionId: scheduleVersionId }),
  patchHistoryQuery: (patch) =>
    set((state) => ({ historyQuery: { ...state.historyQuery, ...patch } })),
  upsertImportJob: (job) =>
    set((state) => ({
      importCenter: {
        ...state.importCenter,
        jobsById: { ...state.importCenter.jobsById, [job.id]: job },
      },
    })),
  setImportCurrentJob: (jobId) =>
    set((state) => ({
      importCenter: { ...state.importCenter, currentJobId: jobId },
    })),
  setRealtimeConnectionStatus: (status) =>
    set((state) => ({
      realtimeBoard: { ...state.realtimeBoard, connectionStatus: status },
    })),
  mergeRealtimeVehicles: (items) =>
    set((state) => {
      const next = { ...state.realtimeBoard.vehiclesById };
      for (const item of items) next[item.vehicleId] = item;
      return {
        realtimeBoard: {
          ...state.realtimeBoard,
          vehiclesById: next,
          lastSyncAt: new Date().toISOString(),
        },
      };
    }),
  archiveCurrentTrip: (actualArrivalAt) =>
    set((state) => {
      if (!state.currentDuty) return state;
      return {
        currentDuty: {
          ...state.currentDuty,
          status: "ARCHIVED",
        },
        historyQuery: {
          ...state.historyQuery,
          tripId: state.currentDuty.tripId,
          trainNo: state.currentDuty.trainNo,
          scheduleVersionId: state.currentDuty.scheduleVersionId,
          date: actualArrivalAt.slice(0, 10),
          source: "attached-route",
        },
      };
    }),
  resetDutyContext: () =>
    set({
      currentDuty: undefined,
      selectedTripId: undefined,
      selectedTrainNo: undefined,
    }),
}));
```

设计原则：

- 当前操作人和当前值乘上下文必须全局唯一
- 历史页筛选条件必须能被多个模块共用
- 实时车位数据只缓存最近状态，不在前端保存无限历史
- 大列表和分页查询不要塞进 Zustand，交给 Query 缓存

## 7. “一回”生命周期状态机

```txt
PLANNED -> ACTIVE -> ARRIVING_TERMINAL -> ARCHIVED
PLANNED -> CANCELLED
ACTIVE -> CANCELLED
```

触发规则：

- 列车发车或司机确认接车后进入 `ACTIVE`
- 实时定位到达终点折返线或调度员手动确认后进入 `ARRIVING_TERMINAL`
- 终点站作业完成后归档到 `ARCHIVED`
- 归档时写入 `trip_history`、`trip_events`，并清空当前值乘上下文

## 8. 统一文件上传与多格式解析服务

核心思路不是“上传接口直接解析”，而是“上传 -> 建任务 -> 解析 -> 标准化 -> 人工复核 -> 入库”。

统一输出一个标准 JSON，所有解析器最终都往这个结构靠拢：

```ts
interface NormalizedImportDocument {

  trains: Array<{
```

### 8.2 解析编排器伪代码

```ts
class ImportParseWorker {
  async handle(jobId: string) {
    const job = await importJobRepo.mustFind(jobId);
    await importJobRepo.update(jobId, { status: "PARSING" });

    const fileBuffer = await objectStorage.read(job.storageKey);
    const parser = parserFactory.create(job.sourceType);

    try {
      const extracted = await parser.extract(fileBuffer, {
        fileName: job.fileName,
      });

      const normalized = await normalizeImportDocument(extracted);
      const validated = await validateNormalizedDocument(normalized);
      const confidence = scoreDocumentConfidence(validated);

      await parsedDocumentRepo.save(jobId, validated);

      const nextStatus =
        confidence < 0.85 || validated.warnings.length > 0
          ? "REVIEW_REQUIRED"
          : "NORMALIZED";

      await importJobRepo.update(jobId, {
        status: nextStatus,
        confidenceScore: confidence,
        parserName: extracted.meta.parserName,
        warnings: validated.warnings,
      });

      if (nextStatus === "NORMALIZED") {
        await importDomainService.confirmAndImport(jobId);
      }
    } catch (error) {
      await importJobRepo.update(jobId, {
        status: "FAILED",
        errors: [serializeError(error)],
      });
    }
  }
}
```

### 8.3 解析器工厂伪代码

```ts
interface SourceParser {
  extract(file: Buffer, ctx: { fileName: string }): Promise<NormalizedImportDocument>;
}

class ParserFactory {
  create(sourceType: ImportSourceType): SourceParser {
    if (sourceType === "XLSX") return new XlsxScheduleParser();
    if (sourceType === "DOCX") return new DocxOcrHybridParser();
    if (sourceType === "PDF") return new PdfOcrHybridParser();
    throw new Error(`unsupported source type: ${sourceType}`);
  }
}
```

### 8.4 Excel 解析器伪代码

```ts
class XlsxScheduleParser implements SourceParser {
  async extract(file: Buffer): Promise<NormalizedImportDocument> {
    const workbook = readWorkbook(file);
    const template = detectWorkbookTemplate(workbook);
    const rows = extractRowsByTemplate(workbook, template);

    const trains = [];
    const circulationSegments = [];

    for (const row of rows) {
      const mapped = mapXlsxRow(row, template);
      if (mapped.type === "TRAIN_STOP") mergeIntoTrain(trains, mapped);
      if (mapped.type === "CIRCULATION_SEGMENT") circulationSegments.push(mapped.segment);
    }

    return {
      meta: buildMeta("XLSX", "xlsx-template-parser"),
      trains,
      circulationSegments,
      dutyAssignments: [],
      warnings: collectXlsxWarnings(rows),
      rawBlocks: [],
    };
  }
}
```

### 8.5 Word / PDF 解析器伪代码

```ts
class DocxOcrHybridParser implements SourceParser {
  async extract(file: Buffer): Promise<NormalizedImportDocument> {
    const textBlocks = await extractDocxTextBlocks(file);
    const tables = await extractDocxTables(file);
    const raw = mergeTextAndTableBlocks(textBlocks, tables);
    return normalizeSemiStructuredBlocks(raw, "docx-hybrid-parser");
  }
}

class PdfOcrHybridParser implements SourceParser {
  async extract(file: Buffer): Promise<NormalizedImportDocument> {
    const textLayer = await tryExtractPdfText(file);
    if (isTextLayerUsable(textLayer)) {
      return normalizeSemiStructuredBlocks(textLayer.blocks, "pdf-text-parser");
    }

    const pageImages = await renderPdfPages(file);
    const ocrBlocks = [];

    for (const pageImage of pageImages) {
      const pageText = await ocrAdapter.recognize(pageImage);
      ocrBlocks.push({ page: pageImage.page, text: pageText, blockType: "OCR" });
    }

    return normalizeSemiStructuredBlocks(ocrBlocks, "pdf-ocr-parser");
  }
}
```

### 8.6 半结构化文档标准化伪代码

```ts
async function normalizeSemiStructuredBlocks(
  rawBlocks: Array<{ page?: number; text: string; blockType: "TABLE" | "PARAGRAPH" | "OCR" }>,
  parserName: string
): Promise<NormalizedImportDocument> {
  const cleaned = rawBlocks.map(cleanTextBlock);
  const candidateLines = splitBlocksToLines(cleaned);

  const trains = extractTrainChains(candidateLines);
  const circulationSegments = extractCirculationSegments(candidateLines);
  const dutyAssignments = extractDutyAssignments(candidateLines);
  const warnings = collectAmbiguities(candidateLines, trains, circulationSegments);

  return {
    meta: {
      sourceType: parserName.includes("pdf") ? "PDF" : "DOCX",
      parserName,
      fileName: "",
      extractedAt: new Date().toISOString(),
      confidenceScore: 0,
    },
    trains,
    circulationSegments,
    dutyAssignments,
    warnings,
    rawBlocks,
  };
}
```

```ts
class ImportDomainService {
  async confirmAndImport(jobId: string) {
    const doc = await parsedDocumentRepo.mustFind(jobId);

    await db.tx(async (trx) => {
      const scheduleVersion = await scheduleVersionRepo.createFromImport(doc.meta, trx);

      for (const train of doc.trains) {
        await masterScheduleRepo.upsertTrain(train, scheduleVersion.id, trx);
      }

      for (const segment of doc.circulationSegments) {
        await circulationRepo.upsertSegment(segment, scheduleVersion.id, trx);
      }

      for (const duty of doc.dutyAssignments) {
        await dutyAssignmentRepo.upsert(duty, scheduleVersion.id, trx);
      }

      await importJobRepo.update(jobId, {
        status: "IMPORTED",
      }, trx);
    });
  }
}
```

## 9. OCR / LLM 预留接口

建议把 OCR 和 LLM 都做成适配器，不直接写死在解析主流程里：

```ts
interface OcrAdapter {
  recognize(image: Buffer): Promise<string>;
}

interface DocumentSemanticAdapter {
  extractStructuredData(textBlocks: string[]): Promise<Partial<NormalizedImportDocument>>;
}
```

使用规则：

- 先走确定性解析：Excel 模板映射、PDF 文本层提取、Word 表格提取
- 再走 OCR
- 最后才走 LLM 语义补全，且结果必须回到统一 JSON，再走校验器

## 10. 最关键的实现建议

- 第一阶段先把 `XLSX -> JSON -> 入库` 跑通，这是最稳定的数据源
- Word/PDF 不要追求一步到位全自动，先支持“解析结果预览 + 人工修正 + 确认入库”
- 历史车次查询一定带 `scheduleVersionId`，否则运行图变更后会串数据
- 历史页统一接受 `tripId/trainNo/routeId/scheduleVersionId/source` 查询参数
- 到达终点归档要同时支持实时事件触发和人工补录，不能只依赖定位

先按 React + Zustand + TanStack Query + NestJS + PostgreSQL + Redis 这条线落地，优先完成 `XLSX 导入、实时车位推送、随车交路归档、历史查询` 四条主链路。
